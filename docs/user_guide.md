# 用户使用手册

## 1. 启动

### GUI 模式（推荐）

```bash
conda activate mandarin-emo-stim
python main.py
```

首次启动会自动检测硬件、检查并下载模型（约 6GB，国内网络走 ModelScope + hf-mirror.com），完成后进入就绪状态。

### 无头模式（分析单个音频文件，不启动 GUI）

```bash
python main.py --headless path/to/audio.wav
# 或
python -m src.cli --audio path/to/audio.wav --out stim.wav
```

## 2. 界面操作

界面采用构成主义风格，分五大块面：

| 块面 | 颜色 | 功能 |
|------|------|------|
| 左侧 | 红色 | 音频输入控制（上传音频、设备选择、录音质量警告） |
| 中部 | 白色 | 核心情感量化展示（NEGATIVE / VALENCE / AROUSAL + 象限 + ASR 文本） |
| 右侧 | 蓝色 | 多模态分解详情（6 模态分项 + 副语言事件） |
| 底部 | 黄色 | 声刺激生成与波形（生成/播放/暂停/停止/保存 + 波形 + 音量） |
| 状态 | 黑色 | 运行状态、推理模式、模型加载进度 |

### 操作流程

1. **输入音频**：点击「上传音频」选择 WAV/MP3/FLAC 文件。
2. **自动分析**：系统自动执行 VAD → ASR → 6 模态特征 → 加权融合。
3. **查看结果**：中部显示三大指标与象限，右侧显示各模态贡献。
4. **生成刺激**：点击「生成刺激波形」，根据当前情绪合成差异化声刺激。
5. **播放/保存**：▶ 播放 / ⏸ 暂停 / ■ 停止 / 💾 保存 WAV。

## 3. 三大核心指标

| 指标 | 含义 | 范围 |
|------|------|------|
| **Negative Score** | 综合负面分，0=极度正面，1=极度负面 | [0,1] |
| **Valence** | 效价 = 1 − Negative，0=负面，1=正面 | [0,1] |
| **Arousal** | 唤醒度，0=平静/低落，1=激动/兴奋 | [0,1] |

## 4. 情绪象限

基于 Russell 情绪环模型：

| 象限 | Valence | Arousal | 含义 |
|------|---------|---------|------|
| Q1 | 高 | 高 | 积极/兴奋 |
| Q2 | 低 | 高 | 焦虑/紧张 |
| Q3 | 低 | 低 | 低落/抑郁 |
| Q4 | 高 | 低 | 放松/满足 |

不同象限会生成不同参数的声刺激（基频、脉冲率、响度、谐和结构等）。

## 5. 声音安全

- 生成刺激的峰值响度限制为 **-10 dBFS**（约 70-75 dB SPL，正常交谈音量），无听力损伤风险。
- 建议使用耳机获得最佳体验（非必须，扬声器同样安全）。
- **特殊人群**（癫痫史、严重心脏病、重度抑郁症正在接受治疗者）建议在专业人员指导下使用。
- 本工具为科研探索用途，**不构成医疗建议或治疗手段**。

## 6. 历史记录

每次分析结果自动存入 SQLite 数据库（`portable_data/history/records.db`）。

- 上限 **200 条**，满额时会弹窗引导导出并清空。
- 通过「文件 → 导出历史记录」可导出 JSON 或 CSV（UTF-8-BOM，Excel 友好）。
